01

Resumen del modelo

Clasificación de salud financiera mensual en tres perfiles.

El simulador clasifica la salud financiera mensual del usuario en tres perfiles: Saludable, Ajustado y En riesgo. La clasificación se basa en una sola variable: el margen disponible mensual, expresado como porcentaje del ingreso total.

Perfil
Saludable
Margen ≥ 20% del ingreso mensual
Perfil
Ajustado
0% ≤ Margen < 20% del ingreso mensual
Perfil
En riesgo
Margen < 0% — los gastos superan los ingresos
02

Fórmulas exactas

Definición matemática de cada cálculo del simulador.

2.1 Ingreso mensual total

El usuario ingresa una o varias fuentes de ingreso con periodicidad. Todas se normalizan a mensual:

// Ingreso mensual total
ingresoMensual = Σ (monto_fuente × factor_periodicidad)
Periodicidad Factor de conversión a mensual Razonamiento
Quincenal × 2 Dos quincenas = un mes
Mensual × 1 Sin conversión
Bimestral × 0.5 Un bimestre = dos meses
Semestral × 1/6 ≈ 0.1667 Un semestre = seis meses
Anual × 1/12 ≈ 0.0833 Un año = doce meses

2.2 Porcentaje de cada categoría

Los sliders representan porcentaje del ingreso mensual. El monto en pesos se calcula así:

// Monto en pesos de cada categoría
montoCOP = (porcentaje / 100) × ingresoMensual

// Gastos personalizados "Otros": COP → % para el cálculo
otros_pct = (totalOtrosCOP / ingresoMensual) × 100
ℹ
Los sliders permiten hasta el 300% del ingreso para reflejar situaciones reales donde los gastos superan ampliamente los ingresos (personas con créditos o deudas altas).
CategoríaVariable de estadoTipo de entrada
Gastos vitalesstate.vitalesSlider (% del ingreso)
Deudas y compromisosstate.deudasSlider (% del ingreso)
Gustos y ociostate.gustosSlider (% del ingreso)
Gastos cotidianosstate.cotidianosSlider (% del ingreso)
Futuro / Ahorrostate.futuroSlider (% del ingreso)
Otros (personalizados)state.otros[]Monto en COP → convertido a %

2.3 Margen disponible

// Margen en porcentaje
margen_pct = 100 - vitales - deudas - gustos - cotidianos - futuro - otros_pct

// Margen en pesos colombianos
margen_cop = (margen_pct / 100) × ingresoMensual
⚠
El margen excluye el ahorro programado (Futuro): es lo que sobra después de todos los gastos y del ahorro. El tooltip en la pantalla de diagnóstico aclara esto con la etiqueta "Disponible del ingreso".

2.4 Clasificación del perfil

function calcularPerfil(margen_pct) {
  if (margen_pct <  0)  return 'riesgo'    // gastos > ingresos
  if (margen_pct >= 20) return 'saludable'  // margen holgado
  return 'ajustado'                          // margen estrecho pero positivo
}

Regla 50/30/20

Marco de referencia orientativo para los diagnósticos y alertas.

50%
30%
20%

Gustos / ocio — hasta el 30%

07

Tratamiento de periodicidades

Normalización de montos no mensuales antes de cualquier cálculo.

// Normalización a mensual
montoMensual = montoIngresado × PERIODICIDAD_FACTOR[periodicidad]
Periodicidad Factor Ejemplo de validación
Quincenal × 2 Arriendo $750.000/quincenal → $750.000 × 2 = $1.500.000/mes ✓
Mensual × 1 Sin conversión
Bimestral × 0.5 Bono $2.000.000/bimestral → $2.000.000 × 0.5 = $1.000.000/mes
Semestral × 1/6 Prima $3.000.000/semestral → $3.000.000 × 0.1667 ≈ $500.000/mes
Anual × 1/12 Cesantías $6.000.000/año → $6.000.000 × 0.0833 = $500.000/mes
08

Subcategorías opcionales

Desglose granular de Vitales y Deudas.

El usuario puede desglosar cada rubro en subcategorías. Cuando lo hace, la suma de las subcategorías reemplaza el valor del slider principal en el cálculo.

if (state.vitalesDetalle.length > 0) {
  // Las subcategorías siempre se ingresan en COP mensual
  montoVitales = Σ (detalle.monto) // ∀ detalle ∈ vitalesDetalle
  state.vitales = Math.round(montoVitales / salario × 100)
}
✓
Las subcategorías siempre se ingresan en COP mensual. El slider refleja el nuevo porcentaje calculado y puede superar el valor previo si las subcategorías suman más.
09

Casos límite

Comportamiento definido ante entradas extremas o incompletas.

Caso Comportamiento esperado
Salario = 0 Los porcentajes no se calculan. Los displays muestran —.
Margen muy negativo (< −100%) calcularPerfil() retorna 'riesgo'. No hay límite inferior para el margen.
Subcategorías > slider original El slider se actualiza automáticamente al nuevo porcentaje (puede superar el valor anterior).
Gastos totales > 300% del ingreso El slider se topa en 300%. Los montos en COP pueden seguir siendo más altos si el usuario los ingresa manualmente.
Una fuente de ingreso en cero Se ignora en el cálculo del total.
Sin fuentes de ingreso registradas El botón "Continuar" en la pantalla de ingresos permanece deshabilitado.
