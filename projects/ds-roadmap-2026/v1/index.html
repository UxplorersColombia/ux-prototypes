Origen del repo
Fork · Azure DevOps
Decisión clave: Figma se abandona progresivamente. El código es la fuente de verdad. Todo el branding Skandia debe fluir a través de la capa tokens → preset usando definePreset, nunca con valores hex hardcodeados en SCSS.
2

Jerarquía de personalización

En este orden de preferencia — no saltarse niveles

1
Preset tokens · definePreset

Cubre ~80% del branding. Primer recurso siempre. Modifica colores, tipografía, radii, sombras y estados en skandia-default.ts.

~80% cobertura
2
Pass Through · pt

Atributos y clases CSS para edge cases que el preset no cubre. No toca lógica de componente.

Edge cases
3
Wrapper · sk-*

Componente Angular propio que encapsula PrimeNG, simplifica la API y fija defaults Skandia. Solo cuando el pt no alcanza.

API simplificada
4
Fork · Copiar fuente PrimeNG

Último recurso absoluto. Genera deuda de mantenimiento y rompe actualizaciones automáticas.

Último recurso

Brechas de madurez detectadas

8 gaps identificados en el estado actual del sistema

ID Brecha Área Severidad
G1 Fork ↔ Azure DevOps sync sin definir — flujo bidireccional inexistente Fundacional 🔴
G2 Fuentes Montserrat / Open Sans no se cargan en ningún punto del repo — bug silencioso que rompe el branding fuera de las máquinas del equipo Fundacional 🔴
G3 0% de cobertura de tests — Vitest instalado pero sin un solo test escrito Calidad 🟠
G4 Accesibilidad ad-hoc, no sistemática — sin auditoría ni criterios WCAG definidos Calidad 🟠
G5 Dark mode funciona en PrimeNG pero no en los componentes sk-* Calidad 🟠
G6 Sin primitivas de layout (grid, stack, container) — bloquea el vibe coding Escala 🟡
G7 Tokens de motion inconsistentes — 0.15s / 0.2s / 0.3s hardcodeados en distintos componentes Escala 🟡
G8 Strings de UI hardcodeados en español — no bloquea, pero dificulta una futura internacionalización Escala 🟡

Fases de implementación

5 fases + Fase 2.5 de iconos

1
Fase 1 · Fundación

Consolidar la capa de tokens

  • Extender tokens más allá de colores: tipografía, spacing, radii, bordes, sombras, estados
  • Cargar fuentes Montserrat / Open Sans correctamente (resuelve G2)
  • Centralizar tokens de motion (resuelve G7)
2
Fase 2 · Calidad

Fortalecer los 10 componentes actuales

  • Mover SCSS hardcodeado al preset skandia-default.ts
  • Auditoría de accesibilidad WCAG 2.1 AA (resuelve G4)
  • Dark mode en todos los sk-* mediante tokens (resuelve G5)
  • Primeros snapshots Playwright como baseline visual (inicia G3)
2.5
Fase 2.5 · Iconos

Estandarizar sistema de iconos

  • Adoptar PrimeIcons como estándar único
  • Eliminar SVGs manuales y Font Awesome del sistema
  • Documentar el catálogo de iconos disponibles
3
Fase 3 · AI-Friendly

Hacer el repo usable por IA

  • Crear CLAUDE.md con instrucciones para agentes de código
  • Specs de componentes (propiedades, variantes, ejemplos)
  • Generar llms.txt como índice para modelos de lenguaje
4
Fase 4 · Escala

Primitivas de layout

  • Crear componentes sk-grid, sk-stack, sk-container
  • Necesarios para vibe coding productivo (resuelve G6)
5
Fase 5 · Proceso

Proceso repetible para nuevos componentes

  • Reemplazar el handoff manual de Figma con un proceso code-first
  • Checklist de contribución: preset → pt → wrapper → (fork como último recurso)
  • Plantilla de spec y criterios de aceptación
5

Secuencia recomendada

Orden de ejecución considerando dependencias y riesgo

1
Foundation — Fase 1

Tokens completos + fuentes (G2) + motion (G7). Base obligatoria antes de todo lo demás.

2
Acuerdo de sincronización con Azure DevOps (G1)

Definir el rol del fork antes de invertir más: ¿laboratorio contribuyente o espejo? Decisión pendiente con el equipo de desarrollo.

3
Cleanup + calidad — Fase 2

SCSS → tokens, snapshots Playwright (G3), dark mode (G5), accesibilidad (G4).

4
Iconos — Fase 2.5

Estandarizar PrimeIcons. Puede solaparse con Fase 2.

5
Scale — Fases 3 y 4

Docs AI-friendly + primitivas de layout (G6). Habilita vibe coding y contribuciones externas.

6
Proceso — Fase 5

Proceso repetible para nuevos componentes. Reemplaza el handoff manual de Figma.

↓
Diferido — G8 Internacionalización

Centralizar strings en español cuando sea necesario. No bloquea ninguna otra fase.
